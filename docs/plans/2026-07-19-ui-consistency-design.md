# UI 一致性修复设计

## 目标

本轮采用“系统一致性修复”方案，不重做信息架构。重点解决：

- 顶部 Tab 的焦点框、选中指示条和文字视觉中心不一致。
- 播放器默认不显示点赞、投币、收藏，升级用户也看不到新入口。
- 详情页社交操作只有图标，发现性较弱。
- 本轮触及的按钮缺少准确的无障碍说明。

## 交互与视觉规范

### 顶部 Tab

- “选中”继续使用品牌色底部短指示条，表达当前栏目。
- “焦点”只包裹文字标签，使用统一的圆角焦点面，不再铺满整个 Tab 容器。
- 标签使用对称的水平、垂直内边距，文字始终位于焦点面的光学中心。
- 缩放仍由 `UserScaleTabLayout` 统一处理；大字号放不下时保持现有自动滚动策略。

### 社交操作

- 播放器 OSD 的默认按钮加入点赞、投币、收藏，顺序放在 UP 主入口之后、列表入口之前。
- 对已有配置执行一次迁移，补齐三个社交按钮；迁移完成后继续尊重用户之后的自定义。
- 详情页保留现有三个操作，补充稳定的可读标签/无障碍语义，不改变现有点击和长按逻辑。

### 一致性边界

- 本轮统一顶部 Tab、详情操作和播放器 OSD 的圆角、间距、焦点反馈与描述文字。
- 不在本轮机械替换全项目所有硬编码尺寸；未触及页面留到后续独立重构，避免一次发布引入大面积回归。

## 验收

- 单元测试覆盖默认集合、旧配置迁移、幂等性和用户后续自定义。
- 主题资源检查、Debug 构建通过。
- TV 模拟器逐屏验证首页 Tab、详情页操作和播放器 OSD；焦点框与文字居中，社交按钮可见且可聚焦。
